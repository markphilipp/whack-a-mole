5 · Claude bonks the mole

Runs --print --permission-mode auto using prompts/bugbot-comment.md or ci-failure.md.

The three modes (set mode: in config.yaml — hot-reloads)

alpha

Detect & log only. Stops at gate in step 3. Run a few days to confirm it sees the right moles.

beta

Full dispatch + Claude edits, but stops before push/reply. Worktree kept for inspection.

live

End-to-end: pushes the fix and replies. The CI loop (step 6→3) is intentional, bounded by the cap.

Why it can't spin forever

  • Bugbot cap — max_bugbot_fixes_per_pr (2), counted from commit trailers
  • CI cap — max_ci_fix_attempts (10); the loop is intentional but bounded
  • Dedupe — tuple cursor + seen maps in state.json
  • Re-run guard — head commit's CI-Auto-Fix: trailer suppresses the same check@sha
  • Normalized keys — matrix/env suffixes stripped so equivalent failures collapse
  • Cap hit — pings the Claude app via PushNotification instead of dispatching
